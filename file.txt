# دليل نشر روبوت إشارات الفوركس على Railway

سيساعدك هذا الدليل في نشر روبوت إشارات الفوركس الخاص بك على Railway، وهي منصة تسهل استضافة التطبيقات.

## الخطوة 1: إعداد مستودع GitHub الخاص بك

أولاً، تحتاج إلى مكان لتخزين رمز الروبوت الخاص بك عبر الإنترنت. GitHub مثالي لذلك.

1.  **إنشاء مستودع GitHub جديد:**
    *   انتقل إلى [github.com](https://github.com/) وقم بتسجيل الدخول.
    *   في الشريط الجانبي الأيسر، انقر فوق الزر الأخضر "New" (أو انقر فوق أيقونة "+" في الزاوية العلوية اليمنى وحدد "New repository").
    *   **اسم المستودع:** اختر اسمًا مثل `forex-signal-bot` أو `my-trading-bot`.
    *   **الوصف (اختياري):** أضف وصفًا قصيرًا.
    *   **عام أو خاص:** يمكنك اختيار "Private" إذا كنت ترغب في الاحتفاظ برمزك لنفسك. يمكن لـ Railway الاتصال بالمستودعات الخاصة.
    *   **تهيئة هذا المستودع بـ:**
        *   حدد "Add a README file".
        *   حدد "Python" لـ "Add .gitignore". سيؤدي هذا تلقائيًا إلى إنشاء ملف يخبر Git بتجاهل الملفات المؤقتة الشائعة المتعلقة بـ Python، وهي ممارسة جيدة.
    *   انقر فوق "Create repository".

2.  **نسخ ملفات الروبوت الخاص بك إلى المستودع الجديد:**
    الآن تحتاج إلى نقل رمز الروبوت من مساحة العمل الحالية إلى مستودع GitHub الجديد هذا.

    *   **الملفات والمجلدات الهامة التي يجب تضمينها:**
        تحتاج إلى تضمين جميع الرموز التي تجعل الروبوت الخاص بك يعمل. فيما يلي قائمة بالملفات والمجلدات الأساسية من مساحة العمل الحالية التي يجب عليك نسخها إلى مستودع GitHub الجديد الخاص بك:

        *   `signal_bot.py` (هذا هو السكريبت الرئيسي الذي يشغل الروبوت الخاص بك)
        *   `test_bot.py` (هذا مفيد لاختبار الإعداد الخاص بك، ولكنه ليس ضروريًا لتشغيل الروبوت على مدار الساعة طوال أيام الأسبوع. من الجيد تضمينه لتصحيح الأخطاء.)
        *   `requirements_bot.txt` (يخبر Railway بالمكتبات التي يحتاجها الروبوت الخاص بك)
        *   `Procfile` (يخبر Railway بكيفية بدء تشغيل الروبوت الخاص بك)
        *   `README_DEPLOY.md` (تعليمات النشر التي راجعناها للتو - مفيدة جدًا!)
        *   `core/` (المجلد بأكمله، يحتوي على التكوين والثوابت الأساسية)
        *   `data/` (المجلد بأكمله، يحتوي على منطق معالجة البيانات، مثل `DataLoader` و `MarketDataEngine`)
        *   `features/` (المجلد بأكمله، يحتوي على جميع منطق هندسة الميزات)
        *   `ml/` (المجلد بأكمله، يحتوي على نماذج التعلم الآلي ومنطق التجميع الخاص بك)
        *   `optimized_results/` (المجلد بأكمله، يحتوي على ملفات `best_config.json` مع إعداداتك المحسّنة لكل زوج)
        *   `models_live/` (قم بإنشاء مجلد **فارغ** باسم `models_live` في مستودع GitHub الخاص بك. سيتم استخدام هذا المجلد بواسطة الروبوت الخاص بك على Railway لتخزين النماذج التي يدربها عند بدء التشغيل الأولي. لا تحتاج إلى وضع أي ملفات *داخل* هذا المجلد في GitHub، فقط المجلد الفارغ نفسه.)

    *   **كيفية النسخ والدفع (باستخدام سطر الأوامر):**
        نظرًا لأنك بالفعل في مستودع Git (`/home/runner/workspace`)، ستحتاج إلى إضافة الملفات إلى هذا المستودع ثم دفعها إلى مستودع GitHub الجديد الخاص بك.

        أولاً، تأكد من إعداد ملف `.gitignore` الخاص بك بشكل صحيح لتجنب دفع الملفات غير الضرورية أو الحساسة عن طريق الخطأ. يجب أن يكون ملف `.gitignore` موجودًا بالفعل من إعدادك الأولي.

        الآن، دعنا نضيف الملفات وندفعها إلى مستودع GitHub الخاص بك:

        ```bash
        # تأكد من أنك في دليل /home/runner/workspace
        cd /home/runner/workspace

        # أضف الملفات والمجلدات الضرورية إلى منطقة التجهيز الخاصة بك
        git add signal_bot.py test_bot.py requirements_bot.txt Procfile README_DEPLOY.md
        git add core/ data/ features/ ml/ optimized_results/
        # أنشئ دليل models_live فارغًا وأضفه
        mkdir -p models_live
        touch models_live/.gitkeep # هذا يحافظ على الدليل الفارغ في git
        git add models_live/

        # قم بتثبيت تغييراتك
        git commit -m "Add Forex Signal Bot files for Railway deployment"

        # الآن، تحتاج إلى دفع هذه التغييرات إلى مستودع GitHub *الجديد* الخاص بك.
        # قد تحتاج إلى تغيير الأصل البعيد لمستودعك الحالي
        # أو الدفع إلى مستودع بعيد جديد. أبسط طريقة للمبتدئين قد تكون
        # إنشاء نسخة جديدة من مستودع GitHub الجديد الخاص بك ونسخ الملفات هناك.

        # لنفترض أنك تريد الدفع من مساحة العمل الحالية إلى مستودع GitHub الجديد الخاص بك.
        # أولاً، قم بإزالة الأصل البعيد الحالي (إن وجد)
        git remote remove origin

        # أضف الأصل البعيد لمستودع GitHub *الجديد* الخاص بك
        # استبدل YOUR_GITHUB_USERNAME و YOUR_REPO_NAME بتفاصيل GitHub الفعلية الخاصة بك
        git remote add origin https://github.com/YOUR_GITHUB_USERNAME/YOUR_REPO_NAME.git

        # ادفع تغييراتك إلى GitHub
        git push -u origin main # أو 'master' إذا كان هذا هو اسم الفرع الافتراضي الخاص بك على GitHub
        ```

        **بديل (أسهل للمبتدئين):**
        إذا كان سطر الأوامر مخيفًا، يمكنك أيضًا القيام بذلك يدويًا:
        1.  على جهاز الكمبيوتر الخاص بك، أنشئ مجلدًا جديدًا.
        2.  انسخ جميع الملفات والمجلدات المدرجة (مثل `signal_bot.py`، `core/`، `data/`، إلخ) من مساحة العمل الحالية إلى هذا المجلد الجديد.
        3.  تأكد من إنشاء مجلد `models_live/` فارغ داخل هذا المجلد الجديد.
        4.  انتقل إلى صفحة مستودع GitHub الجديد الخاص بك.
        5.  انقر فوق "Add file" -> "Upload files".
        6.  اسحب وأفلت جميع الملفات والمجلدات من مجلدك المحلي الجديد إلى منطقة تحميل GitHub.
        7.  قم بتثبيت التغييرات.

## الخطوة 2: النشر على Railway

بمجرد أن يصبح رمزك على GitHub، فإن ربطه بـ Railway أمر مباشر.

1.  **تسجيل الدخول إلى Railway:**
    *   انتقل إلى [railway.app](https://railway.app/) وقم بتسجيل الدخول إلى حسابك.

2.  **إنشاء مشروع جديد على Railway:**
    *   في لوحة تحكم Railway الخاصة بك، انقر فوق الزر "New Project".
    *   حدد "Deploy from GitHub Repo".

3.  **الاتصال بمستودع GitHub الخاص بك:**
    *   سيطلب منك Railway ربط حساب GitHub الخاص بك (إذا لم تكن قد فعلت ذلك بالفعل). قم بتفويض Railway.
    *   سترى قائمة بمستودعات GitHub الخاصة بك. حدد المستودع الذي أنشأته للتو لروبوت إشارات الفوركس الخاص بك (على سبيل المثال، `forex-signal-bot`).
    *   سيكتشف Railway تلقائيًا ملفي `Procfile` و `requirements_bot.txt` ويقترح إعداد خدمة Python. قم بتأكيد ذلك.

4.  **تكوين متغيرات البيئة:**
    هذه خطوة **حاسمة**. يحتاج الروبوت الخاص بك إلى معلومات حساسة (مثل رموز API المميزة) للعمل، ويجب **ألا** تقوم أبدًا بتضمينها في رمزك. يوفر Railway طريقة آمنة لتخزينها كـ "متغيرات البيئة" (Environment Variables).

    *   بمجرد إنشاء مشروعك على Railway، انتقل إلى إعداداته (عادةً عن طريق النقر فوق اسم المشروع، ثم الانتقال إلى علامة التبويب "Variables").
    *   انقر فوق "New Variable" وأضف ما يلي، واحدًا تلو الآخر:

        *   **`OANDA_TOKEN`**:
            *   **الاسم:** `OANDA_TOKEN`
            *   **القيمة:** رمز الوصول الشخصي الخاص بـ OANDA API. (تحصل عليه من إعدادات حساب OANDA الخاص بك).
        *   **`OANDA_BASE_URL`**:
            *   **الاسم:** `OANDA_BASE_URL`
            *   **القيمة:** عنوان URL الأساسي لـ OANDA API. لحساب تجريبي، يكون عادةً `https://api-fxpractice.oanda.com/v3`. للحساب الحقيقي، يكون `https://api-fxtrade.oanda.com/v3`.
        *   **`TELEGRAM_BOT_TOKEN`**:
            *   **الاسم:** `TELEGRAM_BOT_TOKEN`
            *   **القيمة:** الرمز المميز لروبوت Telegram الخاص بك، الذي تم الحصول عليه من BotFather.
                *   **كيفية الحصول على `TELEGRAM_BOT_TOKEN`:**
                    1.  افتح Telegram وابحث عن `@BotFather`.
                    2.  ابدأ محادثة مع BotFather واكتب `/newbot`.
                    3.  اتبع التعليمات لاختيار اسم واسم مستخدم لروبوتك.
                    4.  سيعطيك BotFather رمز HTTP API. يبدو شيئًا مثل `123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11`. انسخ هذه القيمة.
        *   **`TELEGRAM_CHAT_ID`**:
            *   **الاسم:** `TELEGRAM_CHAT_ID`
            *   **القيمة:** معرف الدردشة حيث سيرسل الروبوت الخاص بك الرسائل. هذا رقم يحدد دردشتك مع الروبوت.
                *   **كيفية الحصول على `TELEGRAM_CHAT_ID`:**
                    1.  بعد إنشاء الروبوت الخاص بك باستخدام BotFather، أرسل *أي* رسالة إلى الروبوت الجديد في Telegram (على سبيل المثال، "مرحبًا").
                    2.  افتح متصفح الويب الخاص بك وانتقل إلى عنوان URL هذا (استبدل `<YOUR_BOT_TOKEN>` بالرمز المميز الذي حصلت عليه من BotFather):
                        `https://api.telegram.org/bot<YOUR_BOT_TOKEN>/getUpdates`
                    3.  سترى استجابة JSON. ابحث عن `"chat":{"id":` والرقم بجانبه. هذا هو `TELEGRAM_CHAT_ID` الخاص بك. سيكون رقمًا طويلاً، وربما سالبًا. انسخ هذه القيمة.

5.  **مراقبة النشر والسجلات:**
    *   بمجرد إضافة متغيرات البيئة، سيبدأ Railway تلقائيًا في نشر الروبوت الخاص بك.
    *   انتقل إلى علامة التبويب "Deployments" أو "Logs" في مشروع Railway الخاص بك. سترى عملية البناء ثم إخراج الروبوت.
    *   **ماذا تتوقع:**
        *   **بدء التشغيل الأولي:** سيقوم الروبوت أولاً بإجراء مرحلة تدريب أولية لكل زوج عملات. يتضمن ذلك جلب البيانات التاريخية وتدريب نماذج التعلم الآلي. قد تستغرق هذه العملية بضع دقائق (يقول الموجه أقل من دقيقتين، ولكن قد يختلف ذلك).
        *   **التشغيل:** بعد التدريب الناجح، سيدخل الروبوت في حلقته اللانهائية. كل 5 دقائق، سيتحقق من الإشارات، وإذا تم العثور عليها، سيرسل رسالة منسقة إلى دردشة Telegram الخاصة بك.
        *   **السجلات:** سترى رسائل في سجلات Railway تشير إلى جلب البيانات، وفحص الإشارات، وإرسال رسائل Telegram. إذا كانت هناك أي أخطاء، فستظهر هنا.

هذا كل شيء! بمجرد النشر والتشغيل، سيعمل الروبوت الخاص بك على مدار الساعة طوال أيام الأسبوع، ويتحقق من الإشارات ويرسلها إلى دردشة Telegram الخاصة بك. تذكر أن تراقب سجلات Railway، خاصة خلال الساعات القليلة الأولى، للتأكد من أن كل شيء يعمل كما هو متوقع.
